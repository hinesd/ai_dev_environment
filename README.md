# ClawBot

Dockerized [OpenClaw](https://github.com/openclaw) gateway behind a **Caddy** reverse proxy, using **LM Studio** as the LLM backend.

## Setup

### Prerequisites

- **Docker** (Engine or Desktop)
- **LM Studio** installed, server running with `--bind 0.0.0.0` — see [LLM Backend](#llm-backend)
- GNU Make
- A static LAN IP for the host — configure it in `settings`

### 1. DNS & firewall

Create a local DNS record for `openclaw.home` pointing to your host's LAN IP. With UniFi: **Settings → Routing → Local DNS → A-record**. Otherwise add to each client's `/etc/hosts`:

```
10.20.30.5  openclaw.home
```

Open ports 80/443 to your LAN subnet only. The subnet and port values below match the defaults in `settings`:

```bash
sudo ufw allow from 10.20.30.0/24 to 10.20.30.5 port 80 proto tcp
sudo ufw allow from 10.20.30.0/24 to 10.20.30.5 port 443 proto tcp
sudo ufw allow from 172.30.0.0/16 to any port 1234 proto tcp   # LM Studio (Docker only)
```

### 2. Configure settings

Config lives in two files with a clear boundary:

| File | Committed? | Purpose |
|------|-----------|---------|
| `settings` | Yes | Non-secret defaults: LAN IP, subnets, ports. Edit here for your network. |
| `.env` | No (gitignored) | Secrets only: the gateway password. Auto-generated by `make init`. |

`settings` uses `?=` (set-if-not-set), so values in `.env` always override the defaults. If your LAN IP changes, edit `settings`. If you need to add an API key for a cloud provider, put it in `.env`.

```
# settings  (committed, no secrets)
OPENCLAW_LAN_IP ?= 10.20.30.5
OPENCLAW_DOCKER_SUBNET ?= 172.30.0.0/16
OPENCLAW_LMSTUDIO_PORT ?= 1234

# .env  (gitignored, secrets only)
OPENCLAW_GATEWAY_PASSWORD=<auto-generated>
```

### 3. Deploy

```bash
make init
```

This runs `verify_env → gen_password → down → rebuild → up → wait-healthy → configure`.

### 4. Trust the TLS certificate

Caddy uses a local self-signed CA. Import the root certificate to avoid browser warnings:

```bash
docker compose exec -T caddy cat /data/caddy/pki/authorities/local/root.crt \
  | sudo tee /usr/local/share/ca-certificates/caddy-root.crt
sudo update-ca-certificates
```

On Android: copy `root.crt` to the device, install via **Settings → Security → Encryption & credentials → Install a certificate**.

### 5. Pair your browser

Visit `https://openclaw.home`. Approve the pairing request:

```bash
make devices-list
make devices-approve REQ=<request-id>
```

Click **Reconnect** in the browser. Done once per device.

## LLM Backend

LM Studio is the default provider — it runs locally on the host and exposes an OpenAI-compatible API on port 1234.

### Install & start

1. Download [LM Studio](https://lmstudio.ai) and install it on the host
2. Start the server with bind-to-all-interfaces so Docker can reach it:

```bash
lms server start --bind 0.0.0.0
```

Or toggle **Developer Mode** in the LM Studio UI and set the server to listen on `0.0.0.0`.

### Download a model

The default model is set in `gateway-config.json` under `agents.defaults.model.primary`. Two models are pre-configured — download either one:

| Model ID | Source | Size |
|----------|--------|------|
| `lmstudio/openai/gpt-oss-20b` | LM Studio catalog | 20B |
| `lmstudio/qwen/qwen3.5-9b` | LM Studio catalog | 9B |

In LM Studio, search for the model name (`gpt-oss-20b` or `qwen3.5-9b`) and download the GGUF variant. The provider prefix (`lmstudio/`) is already set in `gateway-config.json` — just provide the model name part to LM Studio.

### Switch models

Edit `services/openclaw/gateway-config.json`, change `agents.defaults.model.primary`, then:

```bash
make configure
```

### Add a model

Add an entry to `models.providers.lmstudio.models` in `gateway-config.json`:

```json
{"id":"provider/model-name","name":"Display Name","reasoning":true,"input":["text"],"contextWindow":131072,"maxTokens":8192}
```

Then `make configure`.

### Use a cloud provider (OpenAI, Anthropic, etc.)

Add the provider's API key to `.env` (secrets file), then add a provider entry to `gateway-config.json`. For example, OpenAI:

```json
{"path":"models.providers.openai","value":{"baseUrl":"https://api.openai.com/v1","api":"openai-completions","auth":"api-key","apiKey":"${OPENAI_API_KEY}","models":[{"id":"gpt-5.4","name":"GPT-5.4","reasoning":true,"input":["text","image"],"contextWindow":131072,"maxTokens":16384}]}}
```

Then `make configure`. See [OpenClaw provider docs](https://docs.openclaw.ai/gateway/configuration#choose-and-configure-models) for more providers.

## Workspace

The agent workspace lives at `workspace/` (mounted to `~/.openclaw/workspace` inside the container). OpenClaw initializes it as a git repo on first run. Keep it in its own private remote — OpenClaw recommends this for backup and portability.

### Back up

```bash
cd workspace
git remote add origin <your-private-repo-url>
git push -u origin main
```

Periodically commit and push new memory, identity, and habit files:

```bash
cd workspace && git add . && git commit -m "memory update" && git push
```

### Restore on a new machine

```bash
rm -rf workspace
git clone <your-private-repo-url> workspace
```

## Daily Commands

```bash
make up           # start
make down         # stop
make restart      # down → up
make logs         # tail gateway + Caddy logs
make qr           # pairing QR for mobile devices
make reset        # full teardown and re-init
```

## Adding Devices

### Browser

Visit `https://openclaw.home` and approve:

```bash
make devices-list
make devices-approve REQ=<request-id>
```

### Mobile (Android / iOS)

1. Install the OpenClaw app
2. Generate a QR code: `make qr`
3. In the app: **Connect → Setup Code → scan the QR**
4. The device auto-pairs. The app will show "checking node capability approval." Approve on the host:
```bash
make nodes-status                         # look for "approval pending"
make nodes-pending                        # prints copy-pasteable command
make nodes-approve REQ=<request-id>
```

Node capabilities must be re-approved whenever the app adds new commands (e.g. location, camera access). `make nodes-pending` surfaces the request ID.

## Configuration

Edit these files, then run `make configure` to apply:

| File | Purpose |
|------|---------|
| `services/openclaw/gateway-config.json` | Gateway settings (models, auth, origins, rate limits) |
| `services/caddy/Caddyfile` | TLS, reverse proxy, `X-Forwarded-User` header |
| `settings` | Non-secret defaults (LAN IP, subnet, LM Studio port) |
| `.env` | Secret: `OPENCLAW_GATEWAY_PASSWORD` (auto-generated) |

Auth is delegated to Caddy (`trusted-proxy` mode). Caddy injects `X-Forwarded-User: openclaw-operator` on every request — no gateway tokens needed. The password in `.env` is only for local CLI operations. Network defaults live in the committed `settings` file.

## How it works

```
LM Studio (host) ← openclaw-gateway (Docker) ← Caddy (Docker, TLS) ← LAN clients
```

Caddy terminates TLS at `https://openclaw.home` and injects a trusted identity header. The gateway's port is never exposed to the LAN. Caddy's source IP falls within the Docker bridge subnet configured in `settings` (`OPENCLAW_DOCKER_SUBNET`), which the gateway trusts. See [OpenClaw docs](https://docs.openclaw.ai) for gateway configuration and agent workspace documentation.
